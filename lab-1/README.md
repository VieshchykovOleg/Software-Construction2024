# Software-Construction2024

1. **DRY (Don't Repeat Yourself)**:
   - Використання інтерфейсів для опису загальних властивостей (`IAnimal`, `IEmployee`, `IFood`) дозволяє уникнути повторень коду.[Див. класи IAnimal,IEmployee,IFood](file:///E:/2_cyrs/2semestr/Software-Construction2024/lab-1/Task1/Task1.cs#L7-L21)

2. **KISS (Keep It Simple, Stupid)**:
   - Чіткі та зрозумілі назви класів і методів спрощують розуміння коду.

3. **SOLID**:[Див. приклади використання SOLID](file:///E:/2_cyrs/2semestr/Software-Construction2024/lab-1/Task1/Task1.cs#L6-43).
   - **S (Принцип єдиної відповідальності)**: Кожен клас (`Animal`, `Employee`, `ZooInventory`, тощо) має лише одну відповідальність. 
   - **O (Принцип відкритості/закритості)**: Код відкритий для розширення (додавання нових типів тварин, співробітників, їжі), але закритий для модифікації, оскільки існуючі класи не потребують змін для додавання нового функціоналу.
   - **L (Принцип підстановки Лісков)**: Використання інтерфейсів для можливості заміни реалізацій без зміни поведінки системи.
   - **I (Принцип розділення інтерфейсу)**: Інтерфейси розділені для визначення конкретних наборів властивостей і поведінки.
   - **D (Принцип інверсії залежностей)**: Залежність від абстракцій (`IAnimal`, `IEmployee`, `IFood`), а не від конкретних реалізацій.

4. **YAGNI (You Aren't Gonna Need It)**:
   - Реалізовано тільки потрібний функціонал, без зайвої складності.

5. **Composition Over Inheritance**:
   - Використання композиції для відносин між класами (`ZooInventory` містить списки тварин, співробітників і їжі) є більш гнучким і уникне обмежень успадкування. [Див. клас ZooInventory](file:///E:/2_cyrs/2semestr/Software-Construction2024/lab-1/Task1/Task1.cs#L53).

6. **Program to Interfaces, not Implementations**:
   - Програмування до інтерфейсів (`IAnimal`, `IEmployee`, `IFood`) спрощує тестування і зберігання, оскільки код працює з абстракціями, а не з конкретними класами. [Див. метод Main](file:///E:/2_cyrs/2semestr/Software-Construction2024/lab-1/Task1/Task1.cs#L93-L114).

7. **Fail Fast**:
   - Використано відповідну обробку помилок, щоб швидко виявляти проблеми та вирішувати. [Див. метод Main](file:///E:/2_cyrs/2semestr/Software-Construction2024/lab-1/Task1/Task1.cs#L83).
